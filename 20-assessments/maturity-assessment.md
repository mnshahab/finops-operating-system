# FinOps Maturity Assessment

## Purpose

This assessment helps organizations evaluate the maturity of their FinOps practice across governance, visibility, allocation, optimization, forecasting, automation, and AI cost management.

The objective is to identify strengths, gaps, and improvement opportunities that increase the business value of technology investments.

---

# Assessment Model

Organizations are assessed across three maturity levels:

| Level | Description                                                |
| ----- | ---------------------------------------------------------- |
| Crawl | Basic visibility and reactive cost management              |
| Walk  | Standardized processes and accountability                  |
| Run   | Optimized, automated, and business-aligned FinOps practice |

---

# Scoring Method

Rate each capability:

| Score | Description            |
| ----- | ---------------------- |
| 1     | Not Implemented        |
| 2     | Partially Implemented  |
| 3     | Implemented            |
| 4     | Consistently Applied   |
| 5     | Optimized and Measured |

# Maturity Classification

## Crawl

Characteristics:

* Limited visibility
* Reactive cost management
* Minimal accountability
* Manual reporting

```text
0 - 80
```

---

## Walk

Characteristics:

* Standardized reporting
* Forecasting process established
* Optimization process active
* Accountability improving


* Executive engagement
